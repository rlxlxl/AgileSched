# Расписание РиМ — расширение YouGile Platform

Telegram-бот для заполнения графика работы в Excel-файле «Расписашка РиМ». Поддерживает типы работы:

| Тип | Цвет | Значение |
|-----|------|----------|
| Полуочно (0,5ч) | `#41DD88` | `0.5` |
| Очно | `#00B050` | `1.0` |
| Дистанционно | `#FF6D01` | `1.0` |

## Требования

- Коробочная версия **YouGile Платформа**
- Node.js (входит в дистрибутив YouGile)
- Telegram-бот (токен от [@BotFather](https://t.me/botfather))
- Excel-файл расписания на диске сервера

## Установка

1. Скопируйте папку `extensions/agile-sched` в `extensions/` коробки YouGile.

2. Установите зависимости:

```bash
cd extensions/agile-sched
npm install
```

3. Положите Excel-файл на сервер, например:

```
/opt/yougile/data/Расписашка РиМ.xlsx
```

4. В YouGile: **Моя компания → Настройки компании → Расширения** — включите «Расписание РиМ».

5. В меню расширения укажите:
   - токен Telegram-бота;
   - абсолютный путь к Excel-файлу.

6. Перезапустите сервер YouGile (расширения стартуют ~15 сек после запуска).

## Использование бота

Команды:

- `/start` — главное меню
- `/schedule` — заполнить расписание
- `/my` — привязать Telegram-профиль к сотруднику
- `/status` — проверить путь к Excel и профиль
- `/cancel` — отменить текущий диалог

Диалог заполнения:

1. Месяц (лист Excel)
2. Неделя (чётная / нечётная)
3. Отдел
4. Сотрудник
5. Начало недели (пн–сб)
6. Конец недели
7. Начальное время
8. Конечное время
9. Вид работы (полуочно / очно / дистанционно)
10. Подтверждение и сохранение

Перед каждой записью создаётся резервная копия в папке `backups/` рядом с Excel-файлом.

## Локальное тестирование (без YouGile)

### Telegram-бот

```bash
cd extensions/agile-sched
npm install
cd ../..

export BOT_TOKEN="токен_от_BotFather"
node scripts/run-bot-local.js
```

По умолчанию создаётся рабочая копия `local-bot-schedule.xlsx` — оригинал не меняется.  
В Telegram напишите боту `/start` и пройдите диалог заполнения.

Свой путь к Excel:

```bash
export EXCEL_PATH="/полный/путь/к/файлу.xlsx"
node scripts/run-bot-local.js
```

### Только парсер / запись

```bash
cd extensions/agile-sched
npm install
cd ../..
node scripts/test-parser.js
```

Скрипт копирует `Расписашка РиМ (1).xlsx` в `test-output.xlsx`, парсит структуру и записывает тестовый диапазон.

## Ограничения

- Работает только в коробке YouGile Платформа (не облако).
- Парсер привязан к текущей структуре Excel-шаблона.
- Не редактируйте Excel одновременно в Desktop Excel и через бота — возможен конфликт.
- Запись защищена mutex — параллельные запросы обрабатываются по очереди.
- Запись идёт через точечный патч xlsx (не полную перезапись), чтобы не ломать файл Excel.

## Поддержка

При изменении шаблона Excel может потребоваться обновить `lib/excel-parser.js`.
